## 六、注意事项

### 6.1 跨域问题

由于浏览器的CORS（跨域资源共享）策略，直接从本地文件访问API可能会被阻止。

**解决方案**:
1. 使用本地HTTP服务器运行
2. 配置服务器允许跨域请求
3. 使用浏览器扩展禁用CORS（仅用于测试）

### 6.2 密钥有效期

- 解密密钥由服务器动态生成
- 每次播放可能获得不同的密钥
- 密钥可能有时效性限制

### 6.3 播放器依赖

需要加载以下资源：
1. **qhwwplayer.js** - QhwwPlayer播放器库（包含FFmpeg WASM）
2. **360_camera_decryptor.js** - 解密工具类
3. **decrypt_fix.js** - 解密修复工具

```html
<!-- 加载 QhwwPlayer 播放器 -->
<script src="qhwwplayer.js"></script>

<!-- 加载解密工具 -->
<script src="360_camera_decryptor.js"></script>
<script src="decrypt_fix.js"></script>
```

### 6.4 浏览器兼容性

需要支持以下特性：
- WebAssembly
- Web Workers
- WebGL
- AudioContext
- Fetch API

## 八、总结

360智能摄像机的视频流加密机制相对简单：
- 使用 `playKey` 作为解密密钥
- 解密在WASM模块中进行
- 通过 `keyType` 指定解密算法类型

通过本报告提供的解密工具，可以：
1. 获取播放信息（包括密钥）
2. 使用密钥解密视频流
3. 正常播放加密的视频流
