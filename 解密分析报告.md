# 360智能摄像机视频流加密解密分析报告

## 一、概述

本报告详细分析了360智能摄像机视频流的加密机制，并提供了相应的解密方案。**本报告仅用于技术研究和学习目的，请勿用于任何非法用途。**

## 二、加密机制分析

### 2.1 视频流获取流程

1. **用户点击预览图片** → 触发播放器初始化
2. **AJAX请求** → 向服务器请求播放信息
3. **返回数据** → 包含视频流地址和解密密钥
4. **播放器加载** → 使用密钥解密并播放视频流

### 2.2 API接口

#### 获取播放信息接口

**接口说明**: 通过摄像机序列号获取视频流播放信息

**请求参数**:
```javascript
{
    taskid: Date.now(),      // 时间戳
    from: '来源标识',        // 来源标识
    sn: '摄像机SN号',        // 摄像机序列号
    mode: 0                  // 播放模式
}
```

**返回数据**:
```javascript
{
    errorCode: 0,            // 错误代码，0表示成功
    errorMsg: '',             // 错误信息
    relayStream: 'xxx',      // 中继流标识
    playKey: 'xxx'           // 播放密钥（加密流才有）
}
```

### 2.3 视频流地址

**格式**: `https://[视频服务器]/[路径]/{relayStream}.flv`

**说明**: 具体的服务器地址和路径由API返回的 `flashUrl` 字段提供。

### 2.4 加密参数

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| key | string | 解密密钥 | null |
| keyType | number | 解密算法类型 | 0 |
| keyForKey | string | 密钥的密钥 | null |
| keyList | array | 密钥列表 | null |

## 三、解密实现

### 3.1 播放器架构

360使用自研的 `QhwwPlayer` 播放器，包含以下模块：

1. **Downloader** - 负责下载视频流数据
2. **Decoder** - 负责解码视频流（包含解密逻辑）
3. **Renderer** - 负责渲染视频画面
4. **UI** - 负责用户界面

### 3.2 解密流程

```
视频流数据 → Downloader下载 → Decoder接收
                                    ↓
                            使用playKey解密
                                    ↓
                            解码为YUV/PCM数据
                                    ↓
                            Renderer渲染显示
```

#### 2. 获取播放信息并设置密钥

```javascript
// 获取播放信息并设置密钥的示例逻辑
$.ajax({
    url: '/api/play',  // 具体接口路径已省略
    data: {
        taskid: $.now(),
        from: '来源标识',
        sn: sn,
        mode: 0
    },
    dataType: 'json',
    success: function(data) {
        if (data && data.errorCode === 0) {
            var videoSrc = data.flashUrl || "默认视频地址"
             
            // 非公开直播使用私密流
            if(ipcData.type != '1'){
                qhwwPlayer.key = data.playKey;  // 设置解密密钥
            }
             
            qhwwPlayer.src = videoSrc;
            qhwwPlayer.play();
        }
    }
});
```

#### 4. 打开解码器（传入密钥）

```javascript
// 打开解码器的示例逻辑
_openDecoder: function() {
    var e = this;
    var t = 0;
    var r = e._opts;
    var n = r.key;           // 解密密钥
    var a = r.keyForKey;     // 密钥的密钥
    var o = r.keyType;      // 解密算法类型
    var i = r.renderType;
    var s = r.keyList;       // 密钥列表
    
    e._opening = 1;
    var u = self.Module;
    var l = u._malloc(28);
    
    // 分配内存并传入密钥
    var c = n ? u.allocateUTF8(n) : null;
    var h = a ? u.allocateUTF8(a) : null;
    var f = s ? u.allocateUTF8(Array.isArray(s) ? JSON.stringify(s) : s) : null;
    
    // 调用WASM打开解码器
    if (0 === (t = u._openDecoder(l, 7, e.videoCbk, e.audioCbk, e.seekCbk, c, o, h, f))) {
        e._opened = 1;
    }
    
    u._free(l);
    e._opening = 0;
    return t;
}
```

## 四、解密工具

### 4.1 工具文件

本项目包含以下文件：

1. **360_camera_decryptor.js** - 解密工具类
2. **decrypt_fix.js** - 解密修复方案（解决画面颜色问题）
3. **qhwwplayer.js** - QhwwPlayer播放器库
4. **index.html** - 视频流解密工具主页面

### 4.2 使用方法

#### 方法1: 使用解密工具类

```javascript
const decryptor = new CameraDecryptor();

// 从API响应提取播放信息
const apiResponse = {
    "errorCode": 0,
    "playKey": "解密密钥",
    "relayStream": "中继流标识",
    "flashUrl": "视频流地址",
    // ... 其他字段
};

const info = decryptor.extractPlayInfo(apiResponse);

// 创建播放器
const container = document.getElementById('video-container');
const player = decryptor.createPlayer(info.videoUrl, info.playKey, container);

// 播放
player.play();
```

#### 方法2: 使用修复版解密工具

```javascript
const decryptor = new CameraDecryptorFixed();

// 从API响应提取播放信息（包含多种配置）

// 选择一个配置创建播放器

const player = decryptor.createPlayer(info.videoUrl, config, container);

// 播放
player.play();
```

## 五、解密配置说明

[`decrypt_fix.js`](decrypt_fix.js:1) 提供了4种解密配置用于解决画面颜色异常问题：

| 配置 | keyType | key | keyForKey | 说明 |
|------|---------|-----|-----------|------|
| 配置1 | 0 | playKey | null | 默认解密方式 |
| 配置2 | 1 | playKey | null | 解密方式1 |
| 配置3 | 0 | null | null | 不使用密钥（测试用） |
| 配置4 | 0 | playKey | relaySig | 使用中继签名作为keyForKey |

## 六、注意事项

### 6.1 跨域问题

由于浏览器的CORS（跨域资源共享）策略，直接从本地文件访问API可能会被阻止。

**解决方案**:
1. 使用本地HTTP服务器运行
2. 配置服务器允许跨域请求
3. 使用浏览器扩展禁用CORS（仅用于测试）

### 6.2 密钥有效期

- 解密密钥由服务器动态生成
- 每次播放可能获得不同的密钥
- 密钥可能有时效性限制

### 6.3 播放器依赖

需要加载以下资源：
1. **qhwwplayer.js** - QhwwPlayer播放器库（包含FFmpeg WASM）
2. **360_camera_decryptor.js** - 解密工具类
3. **decrypt_fix.js** - 解密修复工具

```html
<!-- 加载 QhwwPlayer 播放器 -->
<script src="qhwwplayer.js"></script>

<!-- 加载解密工具 -->
<script src="360_camera_decryptor.js"></script>

```
